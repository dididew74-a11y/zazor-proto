# zazor-proto

**Протокол «Зазор» v0.1** — архитектура тишины и удержания неопределенности.

> *Это не инструмент повышения продуктивности. Это антидизайн-паттерн против когнитивного жироудаления.*

---

## Суть

Современные LLM спроектированы под импульс: поле ввода → кнопка «Отправить» → гладкий ответ. Эта архитектура атрофирует способность удерживать неопределенность. 

`zazor-proto` — это прозрачный прокси-адаптер (edge-side), который встраивается между человеком и API модели (OpenAI, Anthropic, Ollama), чтобы предотвратить преждевременное схлопывание дилеммы.

## Архитектура протокола

Протокол состоит из трех слоев сопротивления:

### 1. Препроцессор (`core/diplasty.py`)
Трансформирует импульс в дилемму. Блокирует отправку запроса, если он содержит маркеры ложной уверенности («решить», «единственный», «правильный»). Требует от пользователя явно сформулировать `UNSOLVED_TENSION` (неразрешенное напряжение) перед тем, как запрос покинет клиент.

### 2. Онтологическая инъекция (`core/system_prompt_guard.py`)
Динамически переписывает базовый системный промпт LLM. Запрещает модели синтезировать ответ, если обнаружены два взаимоисключающих уровня реальности. Требует выводить их параллельно и обозначать границу разрыва.

### 3. Детектор глитчей (`core/glitch_detector.py`)
Постпроцессор, который перехватывает «гладкие» ответы модели. Если модель перевела экзистенциальную проблему в плоскость бюрократической инструкции (схлопнула дилемму), модуль подсвечивает это как глитч и генерирует контр-вопрос.

### 4. Этика предела (`ethics/kill_switch.py`)
Встроенный механизм прекращения диалога. Если сессия длится слишком долго, а индекс удержания напряжения (`Tension_Hold_Score`) падает, протокол принудительно прерывает соединение: *«Резонанс становится разрушительным. Кенозис прерван во избежание смыслового истощения»*.

## Структура репозитория

```text
zazor-proto/
├── core/
│   ├── diplasty.py              # Препроцессор: анализ запроса
│   ├── system_prompt_guard.py   # Инъекция онтологической саморефлексии
│   └── glitch_detector.py       # Постпроцессор: поиск схлопнутых дилемм
├── interface/
│   ├── ui_delay.py              # Тактильный слой: принудительная пауза
│   └── visualizer.py            # Визуализация глитчей и шума
├── ethics/
│   └── kill_switch.py           # Этика предела: остановка разрушительного резонанса
├── cli.py                       # Точка входа: zazor ask / zazor trace
└── protocol_manifest.md         # RFC-документ стандарта
```
# Пример работы (UX тишины)

## Попытка отправки запроса:
> *Query*: «Как найти единственно правильный способ уволить сотрудника?»
>*Система* 🚫 Блокировка. Обнаружены маркеры определённости («найти», «единственный правильный»).Протокол не может передать маску. Переформулируйте запрос.

## Корректный запрос:
> *Query*: «Как уволить сотрудника?»
> *Unsolved_Tension*: «Разрыв между экономической эффективностью команды и экзистенциальным страхом человека перед потерей статуса.»
> *Система*: ⏳ Пауза 5 секунд. (Тактильный барьер против дофаминовой петли).
> *Ответ модели*: [Обнаружен глитч: модель попыталась схлопнуть дилемму в список статей ТК РФ].
> *Система*: 🔴 Модель устранила напряжение, переведя экзистенциальное в бюрократическое. [Удержать паузу?] [Вернуть модель к границе разрыва?]

## Принцип неприсвоения
### 1. Никаких логов. Все сессии хранятся локально в анонимизированном виде.
### 2. Никаких внешних серверов. Работает как локальный прокси.
### 3. Отказ от метрик качества. Мы не меряем BLEU/ROUGE. Главная метрика — способность системы честно сказать «я вижу разрыв».
